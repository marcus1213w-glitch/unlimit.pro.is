IS-339 Feature Design: New Widget for Tailor-Made Pages
DESIGN Medium
Design and implement a widget system that allows users to compose custom dashboard pages by selecting, arranging, and configuring individual widgets (charts, tables, feeds, heatmaps). Each widget is an isolated, self-contained UI module that can be added, removed, resized, and repositioned on a user's personal page layout. Users can save multiple page configurations and switch between them.
Acceptance Criteria
  • Widget catalog panel with searchable list of available widget types (Leaderboard, Token Table, Heatmap, Feed, Signals, Chatbot, Portfolio Summary)
  • Drag-and-drop or click-to-add widget placement on a grid-based page canvas
  • Each widget renders independently with its own data source, refresh interval, and configuration options
  • Widget configuration modal per widget type (e.g., Token Table: select columns; Heatmap: select timeframe; Feed: select filters)
  • Page layout persistence — saved per user with unique page name, restorable on login
  • Multiple page support — users can create, rename, duplicate, and delete custom pages
  • Widget removal via close button or context menu; page resets to default layout option available
  • Responsive layout — widgets reflow on smaller screens (≤1024px stacks to single column)
Tech Notes
Grid Layout (CSS Grid) localStorage Widget Registry Pattern Per-widget Data Isolation

IS-366 New Token Discovery Service
SERVICE High
Build a backend service that continuously monitors the Solana blockchain for newly minted tokens (SPL Token-2022 standard), collects metadata (name, symbol, supply, creator, mint authority), and classifies them by risk level (verified / unverified / suspicious). The service exposes an API endpoint for the frontend to query recently discovered tokens with filtering and sorting capabilities.
Acceptance Criteria
  • Service polls Solana RPC (or Helius DAS API) for new SPL Token-2022 mints at ≤30s intervals
  • Token metadata extraction: name, symbol, decimals, supply, mint authority, creator wallet, creation timestamp
  • Risk classification engine — verified (known creator / audited), unverified (new / no audit), suspicious (honeypot patterns / rug-pull flags)
  • REST API endpoint: GET /api/tokens/discovered with params: limit, offset, risk_level, sort (by_age, by_volume, by_market_cap), search (symbol/name)
  • WebSocket push for real-time new token notifications (payload: token metadata + risk badge + initial liquidity estimate)
  • Data stored in indexed cache (Redis or similar) with TTL — stale entries (>7d without activity) purged
  • Integration with existing Top 100 token pipeline — newly discovered tokens that meet volume/mcap thresholds auto-promoted to Top 100 list
Tech Notes
Solana RPC / Helius DAS SPL Token-2022 REST API WebSocket Push Redis Cache Risk Classification Engine
Dependencies
IS-368 (Discovery Feed UI consumes this service API) · Existing Top 100 token pipeline
IS-368 New Token Discovery Feed & Live Monitor
FEATURE High
Frontend UI that displays a live, auto-updating feed of newly discovered tokens with real-time WebSocket monitoring. The feed shows each new token as a card/row with metadata, risk badge, initial liquidity, and quick-action links (view details, add to watchlist, open in explorer). A sidebar "Live Monitor" panel provides a condensed real-time stream of mint events.
Acceptance Criteria
  • Discovery Feed page — full-page view with sortable, filterable table of newly discovered tokens (columns: age, symbol, name, risk, liquidity, volume, creator)
  • Risk badge per token: ✓ Verified (green), ⚠ Unverified (amber), ✗ Suspicious (red) — clickable to view risk report details
  • Live Monitor sidebar widget — compact scrollable stream showing newest mints in real-time (auto-updates via WebSocket)
  • Filter pills: All / Verified Only / Unverified / Suspicious; sort by: Newest / Highest Liquidity / Most Volume
  • Search input for filtering by token symbol or name
  • Quick actions per token row: Add to watchlist, View token detail, Open Solscan/SolanaFM explorer link
  • Token detail drawer (extends existing token drawer pattern) — shows full metadata, risk report, initial holders, creator wallet link
  • Live Monitor auto-pauses when user scrolls up (resume on scroll-to-bottom or explicit "Resume" button)
  • Feed entries older than 24h collapsed into "Earlier today" section; older than 7d moved to archive view

IS-365 Fund Flow Heatmap Visualization (Top 100 Tokens)
FEATURE High
A heatmap grid visualization showing fund flow intensity for the Top 100 Solana tokens. Each cell represents a token, sized by market cap and colored by net fund flow (inflows = green shades, outflows = red shades, neutral = gray). The heatmap updates in real-time and provides time-range selectors (1h, 4h, 24h, 7d). Hovering a cell shows a tooltip with detailed flow data.
Acceptance Criteria
  • Heatmap grid layout — 10×10 grid (or treemap layout) where each cell = 1 token from Top 100
  • Cell sizing proportional to market cap (treemap style) or equal-size grid cells with mcap indicator
  • Cell coloring: net fund flow direction — inflow gradient (green: oklch 58% 0.16 152 range), outflow gradient (red: oklch 58% 0.20 25 range), neutral (gray/muted)
  • Time range selector pills: 1h / 4h / 24h / 7d — changes the flow calculation window
  • Hover tooltip per cell: token name, symbol, price, 24h change, net flow ($), inflow ($), outflow ($), flow ratio
  • Click on cell opens existing token detail drawer with fund flow section added
  • Legend bar at top explaining color scale (max inflow → neutral → max outflow) with dollar amounts
  • Real-time refresh indicator (live-dot + "WS" badge) — cells animate on data update
  • Responsive: on ≤768px, heatmap switches to a scrollable list view with color-coded rows

IS-367 AI Market Assistant (RAG Chatbot Service)
SERVICE High
An AI-powered market assistant chatbot that uses Retrieval-Augmented Generation (RAG) to answer user questions about market conditions, specific tokens, wallet activities, and news events. The chatbot pulls from on-chain data (elite wallet transactions, token metrics) and news intelligence (AI-enriched news feed) to provide contextual, data-backed responses. Presented as a floating chat panel accessible from any page.
Acceptance Criteria
  • Floating chat button (bottom-right corner) — clickable to expand chat panel overlay
  • Chat panel: message input field, conversation history scroll, suggested prompts, close/minimize button
  • Suggested prompt starters: "What's happening with SOL today?", "Which wallets are accumulating JUP?", "Summarize the latest high-impact news", "Compare WIF vs BONK performance"
  • RAG backend retrieves from: (1) on-chain data — elite wallet PnL, recent transactions, token metrics; (2) news intelligence — AI-enriched news feed items, sentiment data
  • Responses include inline data references (clickable links to token pages, wallet drawers, news items)
  • Response formatting: structured answers with headers, bullet points, data callouts, and source attribution
  • Streaming response display (token-by-token as generated) with typing indicator
  • Context awareness — chatbot knows current page context (e.g., if user is on Token page for SOL, pre-fills context)
  • PRO badge on chat button for Free-tier users — prompts upgrade modal on click
  • Chat history persistence per user session (localStorage or backend)

Elite wallet data · News intelligence feed · Token metrics pipeline · IS-339 (Widget embedding) · PRO subscription system
IS-370 Copy Trading Feature
FEATURE High
A copy trading system that allows users to select elite wallets to follow and automatically mirror their trades (with configurable parameters: allocation amount, max position size, slippage tolerance, enabled/disabled tokens). Includes a portfolio dashboard showing followed wallets, copy trading history, aggregated PnL, and per-wallet performance metrics. Trades are executed via Jupiter Aggregator API with user's connected wallet.
Acceptance Criteria
  • Copy Trading page (replaces existing Portfolio nav) — shows followed wallets list with status, allocation, and PnL per wallet
  • Follow wallet action — from any wallet drawer or leaderboard, "Copy Trade" button adds wallet to followed list
  • Copy configuration per followed wallet: allocation amount (SOL/USDC), max position size, slippage tolerance, token whitelist/blacklist, stop-loss threshold
  • Aggregated PnL dashboard — total copy trading PnL, win rate, number of mirrored trades, active positions
  • Copy trading history table — timestamp, source wallet, token, action (BUY/SELL), amount, your amount, status (completed/failed/pending)
  • Trade execution via Jupiter Aggregator API — automatic swap execution when followed wallet makes a trade
  • Real-time signal → trade pipeline: WebSocket detects elite wallet trade → matches copy config → executes swap via Jupiter → records in history
  • Pause/Resume per wallet — user can pause copy trading for a specific wallet without unfollowing
  • PRO badge — Copy Trading is a Pro feature; Free-tier users see upgrade prompt
  • Risk disclosure banner on Copy Trading page — "Copy trading involves risk. Past performance does not guarantee future results."
Tech Notes
Jupiter Aggregator API WebSocket Signal Pipeline Existing Copy Trading Nav/UI PRO Tier Gate Existing Wallet Drawer
Dependencies
Jupiter Aggregator API · Elite wallet WebSocket signals · Connected wallet (auth flow) · PRO subscription · IS-339 (Widget embedding)
